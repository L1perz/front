## Практическая работа №21 – Кэширование с использованием Redis

### Реализовано:
- REST API на Node.js (Express) для работы с пользователями и товарами
- Аутентификация с использованием JWT (access + refresh токены)
- Ролевая модель доступа (RBAC): `admin`, `seller`, `user`
- Кэширование данных с помощью Redis:
  - `GET /api/users`, `GET /api/users/:id` — кэш 1 минута
  - `GET /api/products`, `GET /api/products/:id` — кэш 10 минут
- Реализован middleware `cacheMiddleware` для автоматической проверки кэша
- Реализована функция `saveToCache` для сохранения данных в Redis
- Реализована инвалидация кэша при изменениях пользователей и товаров

### Проверка работы кэширования:
| Запрос | Результат |
|--------|-----------|
| 1-й `GET /api/products` | `{"source": "server", ...}` |
| 2-й `GET /api/products` (в течение 10 минут) | `{"source": "cache", ...}` |
| `PUT /api/products/1` | кэш сброшен |
| 3-й `GET /api/products` | `{"source": "server", ...}` |

## Практическая работа №22 – Балансировка нагрузки

### Реализовано:
- Backend-сервис на Express, возвращающий свой порт
- **Три экземпляра** backend-сервиса на портах 3000, 3001, 3002
- Балансировка нагрузки с помощью **Nginx**:
  - Алгоритм Round Robin (чередование портов 3000 и 3001)
  - Параметры отказоустойчивости: `max_fails=2 fail_timeout=30s`
  - Резервный сервер: `backend3:3002 backup`

### Проверка балансировки:
| Действие | Результат |
|----------|-----------|
| Несколько запросов к Nginx (порт 80) | Чередование портов 3000 и 3001 |
| Остановка `backend1` и `backend2` | Запросы идут на порт 3002 (резервный) |

## Практическая работа №23 – Контейнеризация приложений с Docker

### Реализовано:
- Создан `Dockerfile` для backend-сервиса (Node.js)
- Настроен `docker-compose.yml` для запуска всех сервисов одной командой
- Контейнеризированы:
  - Три экземпляра backend-сервиса
  - Nginx (балансировщик)
  - HAProxy (альтернативный балансировщик)
  - Redis (кэш)
- Все сервисы объединены в сеть Docker

### Что дала контейнеризация:
- **Воспроизводимость** – приложение работает одинаково на любой системе
- **Изоляция** – каждый сервис в отдельном контейнере
- **Удобство** – вся инфраструктура описывается кодом
